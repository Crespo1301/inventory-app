# Launch Roadmap

## Phase 1: Local Product Test

Goal: prove that the app is faster and more reliable than a whiteboard.

- Build low-stock notes.
- Build inventory item list.
- Build order suggestion logic.
- Export/share the order list.
- Test in one kitchen for one full order cycle.

No account system. No paid backend. No subscriptions.

## Phase 2: Private Mobile Builds

Goal: install on real devices and use it during actual work.

- Configure EAS.
- Create Android preview builds.
- Use Expo Go where possible during early testing.
- Use TestFlight later for iOS testing.
- Capture feedback after every order day.

## Phase 3: Store Prep

Goal: prepare a simple public listing.

- App name and icon.
- Screenshots from real app screens.
- Short description.
- Privacy policy.
- Support email.
- Minimal permissions.
- Internal or closed testing first.

## Phase 4: Paid Product Decisions

Only after product testing:

- Decide free vs. paid app.
- Decide whether restaurants need team sync.
- Decide whether to add subscriptions.
- Decide whether to build a small self-hosted sync backend.

## Cost Reality

- Google Play Console has a one-time developer registration fee.
- Apple Developer Program has an annual membership fee.
- EAS cloud builds are convenient, but local builds or delayed iOS release can keep early costs lower.

The cheapest path is Android-first testing, local-first data, and no backend until the app proves useful.
